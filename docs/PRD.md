# 产品需求文档 — wordmark

> 版本：MVP v1.0
> 状态：内部验证

## 1. 产品定位

**一句话定义**：专为英文阅读打造的"私人词库替换器"，仅在用户主动标记的单词上，将网页中的生词自动内联替换为英文→中文释义，帮助保持阅读流。

**核心价值**：个人词库驱动、无冗余干扰、阅读时直接获取释义。

---

## 2. 用户故事

- 作为中文母语的英语学习者，当我阅读英文网页时，我希望看到我标记过的生词旁边直接显示中文释义，这样我就不用反复查词典。
- 作为用户，当我遇到新的生词时，我希望通过右键菜单快速标记，并让释义自动填充，然后立即在页面中看到替换效果。
- 作为用户，我希望能够管理我的单词本，查看、编辑、删除或导入导出单词。

---

## 3. 功能范围

### 3.1 单词本管理

| 项目 | 说明 |
|------|------|
| 存储位置 | `chrome.storage.local` |
| 数据结构 | `{ id, word, definition, createdAt }` |
| 操作 | 手动添加、右键自动添加、编辑、删除、搜索过滤、显示总词数 |
| 导入/导出 | JSON、CSV 格式 |
| 冷启动 | 预置 CEFR B1-B2 级别 100-200 词示例词库，首次使用可一键导入 |

### 3.2 单词标记方式

- **触发**：选中英文单词 → 右键菜单 "添加到单词本"
- **释义获取**：自动调用翻译 API（英→中），弹出确认窗口供用户修改
- **异常处理**：API 失败时显示错误提示并允许重试

### 3.3 页面替换引擎

- **匹配规则**：全词匹配（`\b` 边界），避免子串误替换；自动将原形词匹配常见屈折变体（`look → looks, looked, looking`）
- **替换范围**：仅处理初始加载的静态文本节点；跳过 `script`、`style`、`textarea`、`input` 等标签
- **替换样式**：`原词(释义)` 格式 + 浅黄色背景高亮
- **空状态**：单词本为空时开启替换无效果，不提示

### 3.4 替换控制

- 默认状态：安装后**不自动启用**
- 三层控制：
  1. 全局开关（Popup 一键切换）
  2. 站点黑白名单
  3. 页面内临时切换（快捷键/悬浮按钮）

### 3.5 扩展弹窗（Popup）

- 当前页命中替换的单词数量
- 全局开关按钮
- 单词本管理入口（展开列表或链接到管理页）
- 导入/导出入口
- 示例词库导入引导（首次空白时）

## 4. 数据模型

```typescript
interface VocabularyEntry {
  id: string;
  word: string;        // 英文原形单词，小写
  definition: string;  // 中文释义
  createdAt: number;   // 时间戳
}

// chrome.storage.local 结构
{
  "vocabList": VocabularyEntry[],
  "settings": {
    enabled: boolean,          // 全局开关，默认 false
    whitelist: string[],       // 白名单域名
    blacklist: string[],       // 黑名单域名
    importSampleDone: boolean  // 是否已导入示例词库
  }
}
```

## 5. 交互流程

### 首次安装
1. 打开任意网页，点击插件图标
2. Popup 显示"单词本为空"，提供"导入示例词库"按钮
3. 用户导入后，单词本包含 100+ 词

### 开启替换
1. 在 Popup 中打开全局开关
2. 当前页面重新执行替换，匹配到的单词变为 `原词(释义)` 带底色

### 标记新词
1. 选中页面单词 → 右键 "添加到单词本"
2. 弹窗显示翻译接口结果，用户确认或修改
3. 保存 → 当前页面及新开的页面立即生效

### 管理词库
1. Popup 点击"管理单词本"
2. 展开列表（搜索/编辑/删除）
3. 导出备份

---

## 6. 非功能需求

- **隐私**：所有数据存储在本地，不上传服务器，不依赖用户登录
- **性能**：500 词内替换延迟 < 200ms（主线程空闲时完成）
- **兼容性**：Chrome 最新两个大版本
- **错误处理**：翻译 API 失败提示重试；页面无文本节点静默跳过；单词本为空无提示
