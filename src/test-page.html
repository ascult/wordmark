wordmark

A browser extension that annotates CET-4 and CET-6 English vocabulary with Chinese translations directly on web pages. Built for Chinese English learners who want to read English content without breaking flow.

Quick Start

git clone https://github.com/anomalyco/wordmark.git
cd wordmark
npm install
npm run build

Load the dist/ folder as an unpacked extension in Chrome or Edge. Enable CET-4 and CET-6 toggles in the popup, then browse any English page. Words from the CET vocabulary are automatically wrapped in highlighted <mark> elements with Chinese definitions.

Features

Core Annotation Engine

  • Multi-pattern matching — Aho-Corasick automaton scans all 5,634 CET words in a single pass
  • Inflection support — automatically matches plural, past tense, gerund, and adverb forms
  • Derivation support — 30 common suffixes resolve derived nouns, adjectives, and verbs back to base CET words
  • Stop-word filtering — 96 high-frequency words excluded from annotation
  • Dynamic content — MutationObserver detects SPA navigation and lazy-loaded content

Translation Pipeline

When CET words are found on a page, the extension sends page segments to Bing Translate for contextual Chinese definitions. The pipeline aligns English and Chinese sentences by index, then extracts the best-matching Chinese word from the pre-built dictionary for each CET entry.

  • Segments are grouped into ~500-character batches for optimal Bing throughput
  • Three concurrent requests per batch with early-stop on consecutive failures
  • Persistent cache stores translations indefinitely (5,000-entry LRU eviction)
  • Fallback uses dictionary cn[] entries when Bing is unavailable

Site Control

  • Enable/disable per domain — whitelist and blacklist with subdomain matching
  • Global toggle — Alt+W keyboard shortcut or popup switch
  • CET-4 / CET-6 independent toggles — choose which vocabulary level to annotate

Vocabulary Management

  • Custom word list — right-click any selected text to add to your vocabulary
  • Import / Export — JSON and CSV formats for bulk vocabulary management
  • Search and edit — full CRUD interface in the options page

Development Notes

Important: The bundled dictionary (dict.json, 1.2 MB) is only included in the service worker bundle. The content script stays lean at ~87 KB. Dictionary lookups happen exclusively in the service worker via message passing.

Common Pitfalls

  1. Async annotation race — Never collect data from DOM marks immediately after calling replaceMatches. Scan document.body.textContent directly instead.
  2. Bing text limit — Real maximum payload is approximately 1,000 characters. Segments are truncated to 950 chars with a trailing ellipsis.
  3. Sentence alignment — Bing may merge or split sentences unexpectedly. The 1:1 index alignment works for most texts, but trailing unmatched sentences are attached to the last aligned sentence.
  4. Word boundaries — Uses Intl.Segmenter for accurate word-boundary detection. A match is only accepted when it aligns exactly with a word segment boundary.
  5. Dictionary fallback — When Bing fails or a word has no contextual match, the system uses pre-extracted Chinese words (cn[] array) from the dictionary, not the full definition.
